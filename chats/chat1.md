## User

[제품 개요]
공장 개인보호구(PPE) 지급·사용기한 자동관리 시스템의 데스크탑 웹 화면을 디자인해줘.
자재창고 출고 데이터를 기반으로 개인보호구 지급대장을 자동으로 최신화하고,
보호구별 사용기한이 임박·경과하면 알람을 보내, 정비 엔지니어의 수기 대장 관리를 없애는 사내 관리 시스템이야.

[사용 환경]
- 데스크탑 전용 (사내 PC, 가로 와이드 화면 기준 1440px 이상). 모바일/반응형은 고려하지 않아도 됨.

[사용자]
- 정비 엔지니어(담당 엔지니어, 주 사용자) : 부서원 지급대장을 관리하고 만료를 사전에 파악
- 현장 작업자 : 본인이 받은 보호구·사용기한 확인
- 자재창고 담당자 : 출고 처리(별도 입력 없이 자동 연동)

[디자인 원칙 — 사내 CMMS(설비관리시스템) 스타일]
첨부 이미지 같은 한국 기업용 설비관리 대시보드 톤을 따른다. 정보 밀도가 높고 기능 위주이며,
한 화면에 여러 데이터 카드/표를 배치한다.
- 레이아웃: 좌측 고정 사이드바(약 240px, 아이콘 + 메뉴명, 펼침/접힘) + 상단 바(탭, 사용자명) + 메인 영역은 2단 그리드의 카드 구성.
- 컬러: 화이트(#FFFFFF) 배경, 라이트 그레이(#F2F2F2) 테이블 헤더, 다크 그레이(#333) 본문.
  메인 액센트는 오렌지(#F5821F 계열) — 활성 탭, 섹션 제목 앞 ▶ 마커, 주요 버튼/강조에만 사용.
- 섹션 카드: 각 카드 상단에 오렌지 ▶ 아이콘 + 굵은 한글 제목, 우측에는 작은 회색 메타텍스트(기준일/필터).
- 표 중심: 얇은 라이트 그레이 보더, 라이트 그레이 헤더, 필요 시 "목표 / 당월 / 누적"처럼 다단 그룹 헤더 사용.
- 카드 안에서 탭으로 필터링(예: 부서별 탭 전환). 깔끔한 선과 여백, 과한 그림자/장식은 지양.
- 타이포: 맑은 고딕 / Noto Sans KR 계열, 본문 12~13px의 촘촘한 크기.
- 상태 표시 색: 정상=그레이, 임박=오렌지, 경과=레드 — 절제해서 뱃지/점으로만.

[좌측 사이드바 메뉴]
대시보드 · 개인별 지급대장 · 부서별 지급대장 · 사용기한/알람 관리 · 보호구 기준정보 · 설정

[구성 화면]

1) 대시보드 (메인)
- 상단 요약 카드 4개(숫자 강조): 만료 임박 N건 / 만료 경과 N건 / 미수령 N건 / 이번 달 지급 N건.
- "오더진행현황"처럼 부서별 현황 그리드: 행=상태(임박/경과/정상), 열=부서, 셀=건수.
- "최근 지급 내역" 표: 작업자 / 보호구 / 지급일 / 사용기한 / 상태 뱃지.
- 우측에 "오늘의 알람" 리스트(임박·경과 항목).

2) 개인별 지급대장
- 좌측에 작업자 검색/선택, 우측에 선택된 작업자의 지급 이력 표:
  보호구 종류 / 지급일 / 수량 / 만료예정일 / 상태(정상·임박·경과 뱃지) / 출고번호.
- 상단에 사번·이름·부서 요약.

3) 부서별 지급대장
- 부서 탭 전환. 작업자(행) × 보호구 종류(열) 매트릭스, 셀에 최신 지급일/상태 표시.
- 우측 상단에 "엑셀 내보내기" 버튼, 기간 필터.

4) 사용기한 / 알람 관리
- 만료 임박·경과 목록 표(필터: 기간/부서/상태).
- 알람 발송 이력 표: 대상 / 보호구 / 발송일 / 채널 / 사유(임박·경과).
- "알람 기준 설정" 영역: 만료 며칠 전 알람(예 30일·7일), 채널 선택.

5) 보호구 기준정보
- 보호구 종류별 표: 종류명 / 사용기한(개월) / 알람 기준일 / 비고. 행 추가·편집 가능.

6) 설정 / 출고 자동 연동
- 출고 데이터 연동 상태(최근 동기화 시각), 연동 로그 표.

[핵심 인터랙션]
- 출고가 발생하면 지급 이력 1건이 자동으로 표에 추가되는 흐름을 화면에 반영.
- 상태(정상→임박→경과)는 지급일 + 사용기한 기준으로 자동 계산되어 뱃지 색으로 표시.
- 표는 정렬/검색/기간 필터 기본 제공.

[성공 지표(화면에 반영)]
지급 내역 누락 0건, 사용기한 경과 미인지 0건을 목표로, 대시보드에서 "임박/경과"가 가장 눈에 띄게 보이도록.
